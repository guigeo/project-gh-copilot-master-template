# Contribuição

## Fluxo recomendado

1. Crie uma branch pequena.
2. Faça mudanças com escopo claro.
3. Rode lint/testes quando o projeto tiver.
4. Abra PR com descrição objetiva.
5. Peça revisão.

## Padrões

- Código simples.
- Documentação quando mudar uso ou configuração.
- Sem segredos versionados.
