# Instruções para documentação

- Use português brasileiro claro e objetivo.
- Todo README deve ter: objetivo, requisitos, instalação, execução, testes e estrutura do projeto.
- Prefira exemplos reais e comandos copiáveis.
- Documente variáveis de ambiente sem revelar valores reais.
- Mantenha documentação curta; detalhes extensos devem ir para `docs/`.
- Use tabelas apenas quando facilitarem comparação.
